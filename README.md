# claude-workflows

A collection of slash commands for [Claude Code](https://docs.anthropic.com/en/docs/claude-code) that automate the full developer workflow — from issue creation to PR merge.

## What's included

| Command | Description |
|---------|-------------|
| `/commit` | Stage changes and create conventional commits with branch safety checks |
| `/pr` | Create detailed PRs with proper documentation and changelog updates |
| `/fix-issue <number>` | End-to-end issue fix: worktree setup → root cause analysis → TDD → PR |
| `/quick-fix <number>` | Autonomous issue resolution with minimal checkpoints |
| `/create-issue <desc>` | Create well-structured GitHub issues with acceptance criteria |
| `/review-pr <number>` | Review a PR for changelog alignment and merge safety |
| `/review-changes` | Meticulous diff review to catch breaking changes and regressions |
| `/scan-debt [dir]` | Scan codebase for tech debt, code smells, and security concerns |
| `/batch-issues <filter>` | Process multiple issues in parallel using subagents |
| `/drain-issues [options]` | Dependency-aware wave processing — analyzes issue deps, batches independent work, repeats until done |
| `/issue-pipeline <desc\|number>` | Full pipeline: create issue → fix → PR → self-review |
| `/full-review <pr-number>` | Claude ↔ Codex review loop — iterative review until approved |

## Installation

### As user-level commands (available in all projects)

```bash
# Clone the repo
git clone https://github.com/leohermoso/claude-workflows.git

# Copy commands to your Claude Code commands directory
cp claude-workflows/*.md ~/.claude/commands/
```

### As project-level commands (available only in a specific project)

```bash
# From your project root
mkdir -p .claude/commands
cp path/to/claude-workflows/*.md .claude/commands/
```

### Verify installation

Open Claude Code and type `/` — you should see the commands in autocomplete.

## Key design principles

- **Git worktrees for isolation** — `/fix-issue` and `/quick-fix` create worktrees so you can work on multiple issues in parallel without conflicts
- **Root cause over surface fixes** — Commands explicitly warn against z-index hacks, retry loops, and other band-aids. They push for understanding *why* before fixing
- **Test-driven fixes** — Write a failing test first, then implement the fix
- **Conventional commits** — All commit messages follow the [Conventional Commits](https://www.conventionalcommits.org/) spec
- **Changelog awareness** — Commands auto-detect and update `CHANGELOG.md` when present
- **No AI attribution** — Commits and PRs are clean — no "Co-Authored-By" or "Generated by Claude" lines
- **Branch safety** — Every command checks you're not on `main`/`master` before making changes

## Requirements

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI
- [GitHub CLI](https://cli.github.com/) (`gh`) — for issue/PR commands
- Git

## Usage examples

```bash
# Fix a GitHub issue end-to-end
/fix-issue 42

# Quick autonomous fix
/quick-fix 42

# Create an issue and fix it in one shot
/issue-pipeline add rate limiting to the API

# Process all open bugs in dependency-aware waves
/drain-issues label:bug

# Scan for tech debt
/scan-debt src/

# Review your changes before committing
/review-changes

# Full Codex review loop on a PR
/full-review 123
```
